# Werewolf Agent Team

基于 LangGraph 的多 Agent 狼人杀博弈系统。  
系统面向“多智能体协作/对抗”“严格信息隔离”“可观测对局日志”“可配置规则与人物能力画像”这几个核心目标，构建一个能够自主完成狼人杀信息不对称博弈的 Agent Team 平台。

---

## 1. 项目目标

本项目旨在实现一个高完成度的狼人杀 AI 对战系统，支持：

- 多 Agent 自主参与狼人杀对局
- 每个 Agent 拥有独立身份、目标、策略和私有信息
- 基于 LangGraph 实现完整的回合流转与有状态编排
- 基于规则引擎完成夜间行动、白天发言、投票放逐、胜负裁决
- 输出结构化日志，支持全程可观测、复盘与评测
- 支持纯 AI 对战与人机混战
- 支持开局前配置人数、板子、规则细项、Agent 能力画像

本项目不是简单的“LLM 聊天模拟狼人杀”，而是一个：

**规则引擎 + 多 Agent 编排 + 可配置角色能力 + 可视化对局平台**

---

## 2. 核心设计原则

### 2.1 规则与策略解耦
- 游戏规则必须由程序硬编码保证
- Agent 只能在允许的行动空间内做决策
- Agent 不得直接修改规则、跳过阶段或伪造裁决结果

### 2.2 严格信息隔离
- 每个 Agent 只能看到：
  - 公共信息
  - 自己的身份信息
  - 自己历史记忆
  - 自己本轮被允许知道的信息
- 任何 Agent 都不能直接获得完整全局真相状态

### 2.3 全程可观测
- 记录每轮、每阶段、每个 Agent 的输入视图、动作输出、规则结算结果
- 所有关键过程以结构化日志形式保存
- 支持回放、复盘与评测

### 2.4 高可配置性
- 支持不同人数板子
- 支持不同规则选项
- 支持不同 Agent 能力画像
- 支持不同模型与上下文预算分配

## 4. 系统总体架构

系统主要分为以下模块：

### 4.1 Game Session Manager
负责：
- 创建对局
- 读取开局配置
- 初始化玩家、身份、规则、运行参数
- 启动 / 暂停 / 恢复一局游戏

### 4.2 LangGraph Runtime
负责：
- 驱动主流程图
- 调用夜晚子图、白天子图、投票结算节点
- 管理全局状态 `GameState`

### 4.3 Rule Engine
负责：
- 身份分配
- 夜间行动结算
- 白天投票放逐
- 警长机制
- 猎人开枪
- 白痴翻牌
- 胜负判定

### 4.4 Agent Factory
负责：
- 创建不同身份的 Agent
- 注入 PersonaProfile
- 绑定模型档位 / 上下文预算 / 推理预算

### 4.5 View Builder
负责：
- 按玩家身份构建私有可见视图
- 保证严格信息隔离

### 4.6 Log Service
负责：
- 保存事件日志、决策日志、系统日志
- 供前端观战、回放、评测使用

### 4.7 Replay / Evaluation
负责：
- 对局回放
- 胜率统计
- 不同模型 / Prompt / Persona 对比
- 过程评测与结果评测

8.6 白痴

正规12人标准局，每天白天就一轮完整发言 → 然后直接投票放逐，不是两轮发言。

一、标准白天流程（一句话版）

1. 天亮 → 法官报昨夜死讯 → 死者遗言。

2. 一轮顺次发言（警长最后说、归票）。

3. 发言结束 → 直接投票（警长1.5票）。

4. 得票最高者出局 → 遗言 → 入夜。

二、平票才会加一轮（PK）

• 投票平票（两人及以上最高）：

◦ 平票玩家再各发一轮言（其他人不能说）。

◦ 再投一次，仍平票则平安日，无人出局。

三、警长竞选（首日专属）

• 首日天亮先竞选警长：上警玩家一轮发言 → 警下投票选警长。

• 选完警长 → 再报昨夜死讯 → 进入上面的标准白天流程。

四、关键时间线（正规房）

• 发言：每人60–120秒，警长90–150秒。

• 投票：15秒，超时弃票。

• 自爆：狼人发言时可自爆，直接吞发言、入夜。
